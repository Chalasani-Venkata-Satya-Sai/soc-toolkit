# Security Policy

Thank you for helping improve the security of **SOC Toolkit**.

We take security vulnerabilities seriously and appreciate responsible disclosure from the security community.

---

## Supported Versions

| Version | Supported |
|---------|-----------|
| v1.x.x  | ✅ Yes |
| < v1.0.0 | ❌ No |

---

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public GitHub Issue**.

Instead, report it privately using one of the following methods:

- **GitHub Security Advisories** (preferred)
- **Email:** chalasanivenkatasatyasai@gmail.com

Please include the following information:

- A detailed description of the vulnerability
- Steps to reproduce the issue
- Affected version(s)
- Proof of Concept (if available)
- Suggested mitigation (optional)

---

## Response Process

We aim to:

- Acknowledge reports within **72 hours**
- Investigate and validate reported issues
- Keep reporters informed throughout the process
- Release security fixes as quickly as possible
- Credit responsible reporters when appropriate (unless anonymity is requested)

---

## Responsible Disclosure

Please:

- Allow us reasonable time to investigate and fix the issue before public disclosure.
- Avoid accessing, modifying, or deleting data that does not belong to you.
- Avoid actions that could negatively affect users or service availability.

We appreciate responsible security research and constructive feedback.

---

## Scope

This policy applies to:

- SOC Toolkit source code
- CLI components
- Streamlit Dashboard
- IOC Enrichment
- Threat Intelligence Integrations
- Phishing Analysis Modules
- YARA Scanner
- Future official releases

Thank you for helping keep **SOC Toolkit** secure.
